Паттерн: Одиночка (Singleton)
-----------------------------------------------------------------
Цель:

Гарантировать, что у класса будет только один единственный
экземпляр и к нему будет предоставлена глобальная точка доступа.
-----------------------------------------------------------------
Зачем:

Для создания единственного экземпляра определенного класса.
-----------------------------------------------------------------
Пример:

- необходим только один экземпляр конкретного класса, который
доступен для всех клиентов;
- единственный экземпляр должен наследоваться путем порождения
подклассов, при этом клиенты имеют возможность работать с
классом-наследником без модификации своего кода.
-----------------------------------------------------------------
