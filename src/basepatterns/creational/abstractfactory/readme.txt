Паттерн: Абстрактная фабрика (Abstract Factory)
----------------------------------------------------------------
Цель:

Создание интерфейса для создания множества взаимосвязанных
или взаимозависимых объектов без жесткой привязки к конкретным
классам.
----------------------------------------------------------------
Зачем:

Для создания множеств взаимосвязанных объектов.
----------------------------------------------------------------
Примеры:

- система не должна зависеть от метода создания, компоновки
и представления входящих в нее объектов;
- входящие взаимосвязанные объекты должны использоваться
вместе;
- система должна конфигурироваться одним из множеств объектов,
из которых она состоит;
- необходимо предоставить множество объектов, раскрывая только
их интерфейсы, но не реализацию.
----------------------------------------------------------------
